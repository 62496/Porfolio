User Roles & Access

USER Role

Basic authenticated user

  • Browse book catalog
  • View book details
  • Manage favorites
  • Follow authors/series
  • Track reading progress
  • Send messages
  • Create collections

AUTHOR Role

Content creators

  • All USER permissions
  • Create new books
  • Edit own books
  • Create author profiles

LIBRARIAN Role

Content moderators

  • All AUTHOR permissions
  • Edit any book
  • Manage reports
  • Resolve issues
  • Dismiss reports

SELLER Role

Inventory managers

  • All USER permissions
  • Manage book inventory
  • Track stock levels
  • Update quantities

// src/features/yourFeature/hooks/useYourFeature.js
import { useState, useEffect, useCallback } from 'react';
import yourFeatureService from '../../../api/services/yourFeatureService';
import { useToast } from '../../../hooks/useToast';

export const useYourFeature = () => {
    // State
    const [items, setItems] = useState([]);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);
    const { showToast } = useToast();

    // Fetch data on mount
    useEffect(() => {
        fetchItems();
    }, []);

    // API calls wrapped in useCallback
    const fetchItems = useCallback(async () => {
        try {
            setLoading(true);
            const response = await yourFeatureService.getAll();
            setItems(response.data);
        } catch (err) {
            setError(err.message);
            showToast('Failed to load items', 'error');
        } finally {
            setLoading(false);
        }
    }, [showToast]);

    const createItem = useCallback(async (data) => {
        try {
            const response = await yourFeatureService.create(data);
            setItems(prev => [...prev, response.data]);
            showToast('Item created successfully', 'success');
            return response.data;
        } catch (err) {
            showToast('Failed to create item', 'error');
            throw err;
        }
    }, [showToast]);

    // Return everything the UI needs
    return {
        items,
        loading,
        error,
        fetchItems,
        createItem,
        // ... other methods
    };
};

        .string()
        .required('ISBN is required')
        .matches(/^\d{13}$/, 'ISBN must be exactly 13 digits'),

    publishingYear: yup

        .max(new Date().getFullYear() + 1, 'Year cannot be in the future')
        .typeError('Publishing year must be a number'),

        .positive('Pages must be positive')
        .integer('Pages must be a whole number')
        .min(1, 'Pages must be at least 1')
        .max(20000, 'Pages cannot exceed 20,000')
        .typeError('Pages must be a number'),

    // Long text validation
    description: yup

    // Array validation (multi-select)

        .array()

        .required('Authors are required'),

    genres: yup
        .array()
        .min(1, 'Please select at least one genre')
        .required('Genres are required'),

    // Optional fields
    seriesId: yup
        .string()
        .nullable()
        .optional(),

    // Conditional validation
    newSeriesTitle: yup
        .string()
        .when('seriesId', {

Multi-Step Form Pattern

For complex forms, split into steps with per-step validation:

// In your form component

const steps = ['Basic Info', 'Authors', 'Genres'];

// Define which fields to validate per step

    0: ['isbn', 'title', 'publishingYear', 'pages', 'description'],
    1: ['authors'],
    2: ['genres']
};

// Validate current step before proceeding
const validateCurrentStep = async () => {
    return await trigger(stepFields[currentStep]);
};

const handleNext = async () => {
    if (await validateCurrentStep()) {
        setCurrentStep(prev => prev + 1);
    }
};

const handleBack = () => {
    setCurrentStep(prev => prev - 1);
};

// In JSX
<ProgressSteps currentStep={currentStep} steps={steps} />

    <div>
        {/* Basic info fields */}
    </div>
)}

{currentStep === 1 && (
    <div>
        {/* Author selection */}
    </div>

{currentStep === 2 && (
    <div>
        {/* Genre selection */}
    </div>
)}

    {currentStep > 0 && (

Form Best Practices

Validation Modes

  • mode: 'onBlur' - Validate when field loses focus
  • mode: 'onChange' - Validate on every change
  • mode: 'onSubmit' - Only validate on submit
  • Use onBlur for best UX

Error Display

  • Show errors after field is touched
  • Use touchedFields.field && errors.field
  • Clear errors when user starts typing
  • Show inline errors below inputs

Submit Handling

  • Disable button when !isValid
  • Show loading state during submit
  • Call reset() after successful submit
  • Handle API errors with toast notifications

Performance

  • Use Controller for controlled inputs
  • Avoid re-rendering entire form on changes
  • Use watch sparingly
  • Split large forms into steps
